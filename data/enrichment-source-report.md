# Player Database Enrichment Report

**Allowed sources only:** Wikipedia, Official Super League website (superleague.co.uk)

**Banned sources:** Rugby League Project, club websites, fan databases, news sites, forums, AI estimates.

Machine-readable details: `data/enrichment-source-report.json` (generated by `npm run enrich:audit`).

## How to re-run

```bash
# Build Super League player index (one-time, ~10–15 min)
npm run enrich:sl-index

# Enrich from Wikipedia + SL index
npm run enrich:players

# Dedupe + enrich + report
npm run enrich:audit

# Remove cross-file duplicates only
npm run audit:duplicates
```

## Source rules

| Player type | Nationality | Apps / Tries | Years active |
|-------------|-------------|--------------|--------------|
| Current | Wikipedia or SL site | SL career stats preferred; Wikipedia club rows as fallback | `YYYY–Present` when still active |
| Historic / retired | Wikipedia | Wikipedia infobox career totals | Wikipedia professional career (`YYYY–YYYY`) |
| Unknown | Left as `Unknown` | Left unset (UI shows **Unknown**) | Left unset |

No guessing. No Rugby League Project. No club or news sites.

## Duplicate handling

`npm run audit:duplicates` merges cross-file duplicates (e.g. Mick/Mickey Higham, Robbie Paul/Robbie Hunter-Paul) using normalized name keys in `src/lib/player-name-normalize.ts`.

## UI display

- **Apps / Tries:** numeric value when confirmed; otherwise **Unknown**
- **Years active:** `2018–Present` for active players when confirmed; otherwise existing value or unknown
- **Nationality:** shown via abbreviation; unknown nationalities remain **Unknown**
